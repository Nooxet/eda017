Klassen Keyboard
import se.lth.cs.pt.io.Keyboard;

Med hjälp av klassen Keyboard kan vi läsa värden av olika slag från tangentbordet. 

    int n = Keyboard.nextInt();              // läser in ett heltal
    double x = Keyboard.nextDouble();        // läser in ett reellt tal
    String word = Keyboard.next();           // läser in ett ord
    String wholeLine = Keyboard.nextLine();  // läser in en hel rad
Dessa operationer finns även i en variant med ledtext: 
    n = Keyboard.nextInt("Ange n: ");
    x = Keyboard.nextDouble("Ange x: ");
    word = Keyboard.next("Skriv in ord: ");
    wholeLine = Keyboard.nextLine("Skriv in en rad med text: ");
Vi kan kontrollera om det finns ett värde av ett givet slag att läsa innan vi försöker: 
    System.out.print("Ange ett heltal: ");
    if (Keyboard.hasNextInt()) {
        System.out.println("Värdet är " + Keyboard.nextInt());
    } else {
        System.out.println("Det fanns inget heltal att läsa!");
        System.out.println("Istället matade du in texten: " + Keyboard.nextLine());
    }
Om användaren skulle mata in en sträng när vi vill läsa in ett tal så kommer vi att få exekveringsfel, vi kan undvika sådana exkveringsfel genom att skriva: 
    System.out.print("Skriv in ett heltal: ");
    while (!Keyboard.hasNextInt()) {
        System.out.print("Du måste skriva in ett heltal, försök igen: ");
        Keyboard.nextLine();
    }
    int value = Keyboard.nextInt();
    System.out.println("Värdet är: " + value);

Klassmetoder
void usePTMode()
Gör att Keyboard-klassen hanterar blandad inläsning av strängar och tal lite enklare än standardklassen Scanner - gör att bland annat nextInt()- och nextDouble()-operationerna inte fungerar riktigt som motsvarande Scanner-operationer.

Följande exempel visar skillnaden: Om vi i ett program först vill läsa in ett heltal, och därefter en sträng, så kan vi med Keyboard-klassen skriva: 

    int key = Keyboard.nextInt("Ange nyckel: ");
    String message = Keyboard.nextLine("Ange meddelande: ");
Om Keyboard-operationerna hade fungerat som motsvarande Scanner-operationer så hade raderna ovan inte fungerat, vi hade varit tvungna att läsa bort radslutet efter inläsningen av talet först: 
    Scanner scanner = new Scanner(System.in);
    int key = scanner.nextInt("Ange nyckel: ");
    scanner.nextLine();   // här måste vi läsa bort radslutet efter talet...
    System.out.print("Ange meddelande: ");
    String message = scanner.nextLine();
Om vi vill att Keyboard-operationerna skall fungera på samma sätt som motsvarande Scanner-operationer så måste vi någonstans i vårt program skriva Keyboard.useScannerMode() (se nedan) - vi måste därefter själva se till att läsa bort radslutet efter inläsning av tal innan vi läser in en sträng. Standardinställningen gör att vi slipper läsa bort radsluten själva, så du behöver inte skriva Keyboard.usePTMode() i ditt program om du inte tidigare i programmet har skrivit Keyboard.useScannerMode(), och nu vill ändra tillbaka.

Observera att vi fortfarande kan läsa flera tal på samma rad, även om vi läser i 'pt-mode' - programmet: 

     System.out.print("Skriv in 5 tal: ");
     double sum = 0;
     for (int i = 1; i <= 5; i++) {
         sum += Keyboard.nextDouble();
     }
     System.out.println("Summan är " + sum);
kan köras på följande sätt: 
     Skriv in 5 tal: 1 2 3 4 5
     Summan är 15.0
void useScannerMode()
Gör att Keyboard-operationerna fungerar precis som motsvarande Scanner-operationer (fast med möjlighet att använda ledtext vid inläsning) - gör att vi kan få problem när vi blandar inläsning av tal och strängar.

Denna inställning gör att vi måste läsa av radslut efter inläsning av tal innan vi läser in en sträng: 

    Keyboard.useScannerMode();
    int key = Keyboard.nextInt("Ange nyckel: ");
    Keyboard.nextLine();    // vi måste läsa in radslutet först
    String message = Keyboard.nextLine("Ange meddelande: ");
boolean hasNext()
Avgör om det finns något mer att läsa.
Returnerar true om det finns mer att läsa, false annars.
String next()
Ger nästa sträng som kan läsas. Strängen avgränsas av mellanslag, tab eller radslut.
String next(String message)
Skriver ut en ledtext, och ger nästa sträng som kan läsas. Strängen avgränsas av mellanslag, tab eller radslut.
Parametrar:
message:ledtexten
String nextLine()
Ger nästa rad med text som kan läsas in.
String nextLine(String message)
Skriver ut en ledtext, och ger nästa rad med text som kan läsas in.
Parametrar:
message:ledtexten

String nextPassword(String message)
Skriver ut en ledtext, och läser in en text som inte skrivs ut på skärmen medan den läses in (det kan vara lämpligt om vi skall mata in exempelvis lösenord). Texten består av alla tecken som skrivs in fram till radslut (dvs då användaren trycker på RETURN-tangenten), och texten returneras som en String.
Parametrar:
message:ledtext
