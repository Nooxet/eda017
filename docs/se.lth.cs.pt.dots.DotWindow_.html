Klassen DotWindow_
import se.lth.cs.pt.dots.DotWindow_;

Observera att du inte skall importera själva fönstret med hjälp av import-satsen ovan, utan istället skall välja någon av klasserna som beskrivs nedan (man kan inte skapa den typ av fönster som import-satsen ovan ger, den finns där av rent tekniska skäl).

Ett DotWindow är ett fönster i vilket användaren kan rita ut 'punkter' (dots) i olika färger (se klasserna Color och Grayscale). Punkternas storlek bestäms när fönstret skapas, som minst kan punkterna vara lika små som skärmens pixlar.

Exempel på användning:

  Color linen = new Color(250, 240, 230);
  Color navy = new Color(0, 0, 128);
  Color gold = new Color(255, 215, 0);
  DotWindow w = new DotWindow(40, 40, 10);
  w.fillWith(linen);
  w.setDot(10, 10, navy);
  for (int x = 0; x < w.getWidth(); x++) {
      w.setDot(x, 5, gold);
  }

Det finns flera olika slags DotWindow-klasser i kurspaketet, det som skiljer dem åt är hur användaren kan styra vad som händer. De har alla samma operationer för att rita saker i fönstret, dessa operationer dokumenteras på denna sida -- för olika sätt att hantera användarens aktiviteter får du se dokumentationen av nedanstående klasser:

  • En klass som bara kan vänta på att användaren klickar i fönstret -- denna klass ligger i se.lth.cs.pt.dots.basic-paketet.

  • En klass som låter oss välja vilka händelser fönstret skall lyssna på, och som kan vänta på att någon av dessa händelser skall inträffa och skicka tillbaka ett särskilt händelseobjekt. Denna klass ligger i se.lth.cs.pt.dots.events-paketet.

  • En klass som låter oss med jämna mellanrum känna av vad som har hänt i fönstret sedan vi senast kollade. Denna klass ligger i se.lth.cs.pt.dots.polling-paketet.

  • En klass som vi kan få att skicka meddelanden till särskilda 'lyssnare' när olika händelser inträffar. Denna klass ligger i se.lth.cs.pt.dots.listeners-paketet.

Operationerna nedan är alltså gemensamma för alla DotWindow-klasser.

Metoder
void fillWith(Color color)
Fyller fönstret med en given färg.
Parametrar:
color:den färg fönstret skall få.
void setDot(int x, int y, Color color)
Sätter ut en prick i given färg, på ett givet ställe i fönstret.
Parametrar:
x:prickens x-koordinat.
y:prickens y-koordinat.
color:den färg pricken skall ha (se klassen Color).
Color getDot(int x, int y)
Ger färgen på en prick med givna koordinater (se klassen Color).
Parametrar:
x:prickens x-koordinat
y:prickens y-koordinat
int getWidth()
Ger antalet prickar i x-led.
int getHeight()
Ger antalet prickar i y-led.
int getDotSize()
Ger storleken på prickarna (i pixlar).
void useCircularDots(Color backgroundColor)
Får fönstret att rita cirkulära prickar (istället för kvadratiska, som är standard).
Parametrar:
backgroundColor:den färg som ritas bakom prickarnas kanter
void useRectangularDots()
Får fönstret att åter rita kvadratiska prickar.
void setAutoUpdate(boolean update)
Slår på eller stänger av automatisk uppdatering av fönstret. Varje omritning av fönstret är ganska tidskrävande, så om vi vill göra många setDot-operationer på en gång kan det vara bra att stänga av den automatiska omritningen och istället begära omritning (med update()) när alla setDot-anrop är klara.
Parametrar:
update:avgör om vi skall ha eller inte ha automatisk uppdatering (standardvärdet är true).
void update()
Gör att hela fönstret uppdateras (användbart om vi stängt av den automatiska uppdateringen).
