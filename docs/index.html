Dokumentation av klasserna i pt-paketet

Observera att kurspaketet är under utveckling, och att det därför kan komma att bli en del tillägg och ändringar i denna dokumentation under kursens gång. På kurshemsidan meddelas så snart en ny version av kurspaketet finns att ladda hem (både jar-fil och dokumentation).

Klasserna i kurspaketet är uppdelade i olika mindre paket (se.lth.cs.pt.io, se.lth.cs.pt.turtle, etc), och det finns exempel på klassnamn som förekommer i flera sammanhang (exempelvis Color som används både till GraphicsWindow och DotWindow, men som ser lite olika ut beroende på i vilket sammanhang klassen används). Dessutom har vi flera olika slags GraphicsWindow- och DotWindow-klasser. Du måste därför titta noga efter att du läser dokumentationen av rätt version av dessa klasser.

Klasserna finns sammanfattade i sina användningsområden längre ner på sidan, men först en lista med samtliga klasser i bokstavsordning:

En sammanställning av olika delpaket

In och utmatning

  • Keyboard: En klass som låter oss läsa från tangentbordet på ett enkelt sätt.
  • IO: Denna klass gör det enkelt för oss att skapa olika in- och utmatningsenheter.
  • ImageReader: Underlättar inläsning av bilder från filer.

GraphicsWindow-klassen

I ett GraphicsWindow kan vi rita

  • sköldpaddor (se nedan), och

  • olika slags figurer (se nedan).

Det finns fyra olika slags GraphicsWindow som vi kan använda för våra sköldpaddor och figurer, de har alla ett antal operationer som beskrivs här. De fyra alternativa fönsterklasserna är:

  • Primitiva fönster: Detta är den enklaste varianten, och de har förutom de vanliga fönsteroperationerna bara operationer för att stanna programmet och vänta på att användaren klickar med musen (se här). Vi importerar dessa fönster med: 

    import se.lth.cs.pt.graphics.basic.GraphicsWindow;

  • Fönster med polling: I dessa fönster kan vi förutom att använda de vanliga fönsteroperationerna även undersöka vad användaren har gjort sedan vi senast tittade efter (se här). Dessa fönster importerar vi med: 

    import se.lth.cs.pt.graphics.polling.GraphicsWindow;

  • Fönster med händelser: Dessa fönster kan vi få att lyssna på olika slags händelser, och vi kan sedan fråga fönstret efter alla nya händelser. Vi kan dessutom utföra de vanliga fönsteroperationerna. Vi importerar dessa fönster med: 

    import se.lth.cs.pt.graphics.events.GraphicsWindow;

  • Fönster med lyssnare: Till detta slags fönster kan vi koppla olika slags lyssnare -- så snart olika händelser inträffar anropas bestämda operationer i våra lyssnare. Vi kan dessutom utföra de vanliga fönsteroperationerna. Vi importerar dessa fönster med: 

    import se.lth.cs.pt.graphics.listeners.GraphicsWindow;

Turtle-klasser

Det finns två Turtle-klasser, de har ungefär samma operationer -- den största skillnaden är att den ena klassen ger sköldpaddor som syns i fönstret. För att importera den osynliga sköldpaddan skriver man: 

    import se.lth.cs.pt.turtle.Turtle;
och för att få den synliga varianten skriver man istället: 
    import se.lth.cs.pt.turtle.visible.Turtle;

  • Turtle - osynlig
  • Turtle - synlig

Färger och figurer

Det finns ett antal operationer som samtliga figur-klasser kan utföra, de beskrivs i klassen Shape nedan, övriga operationer beskrivs i respektive figurklass.

  • Color -- beskriver de färger som vi kan rita i ett fönster.
  • Shape (operationer som är gemensamma för samtliga figurklasser)
    • Rectangle
    • Oval
    • Text
    • Sprite

DotWindow-relaterade klasser

  • DotWindow -- själva fönstret som vi ritar i. Det finns fyra olika slags DotWindow-klasser, det som skiljer dem åt är att de låter oss hantera användarens aktiviteter på olika sätt.
  • Color -- beskriver de färger som vi kan rita i ett fönster.

Template-klasser för spelprogram

Nedanstående två klasser 'kapslar in' en händelseloop, så att du enkelt kan skriva spelprogram i ett GraphicsWindow eller i ett DotWindow.

  • GameTemplate (för GraphicsWindow)
  • GameTemplate (för DotWindow)

Tidtagning och händelser

  • Clock: En klocka som vi kan använda för att stoppa programmet under en given tidsrymd.
  • StopWatch, kan användas för att ta tiden på ditt program.
  • Event: En slags händelse som vi kan utnyttja när vi använder de 'listener'-baserade fönsterklasserna.
  • TimeListener
  • Timer

Diverse övriga klasser

  • CSMath: diverse hjälpoperationer.
  • Die: vanliga speltärningar.
  • MazeGenerator, som kan användas för att generera matriser som beskriver labyrinter (används i labyrint-alternativet på inlämningsuppgift 4).
  • Path, ritar en gubbe som hoppar på stenar i ett fönster.

Javas standardklasser

Javas standardpaket (som innehåller några av de klasser som vi använder i kursen) finns dokumenterat på Javas dokumentationshemsida.

Nedan finns direktlänkar till några av de klasser vi använder i kursen:

  • Math innehåller ett antal användbara matematiska funktioner.
  • Random ger olika slags slumptal.
  • Scanner, som används för att läsa in värden från tangentbord eller fil.
  • String, som beskriver en sträng.
  • StringBuilder, som kan användas för att bygga upp en sträng (String-objekt är oföränderliga).
  • StringTokenizer, som kan användas för att dela upp en sträng i delsträngar.
